## Pagination

Most list endpoints support pagination to efficiently retrieve large datasets. Pagination uses cursor-based or offset-based approaches depending on the endpoint.

## Offset-Based Pagination

Used by endpoints like `GET /v1/numbers` and `GET /v1/conversations`.

| Parameter | Type | Description |
|-----------|------|-------------|
| `limit` | integer | Number of results per page (default: 20, max: 100) |
| `offset` | integer | Number of results to skip (default: 0) |

<CodeGroup>
```javascript Example: Paginated request
// Get first page (20 results)
const page1 = await fetch(`${BASE_URL}/v1/numbers?limit=20&offset=0`, {
  headers: { 'Authorization': `Bearer ${API_KEY}` }
});

// Get second page
const page2 = await fetch(`${BASE_URL}/v1/numbers?limit=20&offset=20`, {
  headers: { 'Authorization': `Bearer ${API_KEY}` }
});
```

```json Response Format
{
  "data": [...],
  "hasMore": true,
  "total": 42
}
```
</CodeGroup>

<Note>
  - `hasMore`: `true` if more results are available
  - `total`: Total number of items (may not be available for all endpoints)
</Note>

## Cursor-Based Pagination

Used by message endpoints like `GET /v1/numbers/:id/messages`. Uses timestamps for efficient pagination of time-ordered data.

| Parameter | Type | Description |
|-----------|------|-------------|
| `limit` | integer | Number of results per page (default: 50, max: 200) |
| `before` | string | ISO timestamp - get messages before this time |
| `after` | string | ISO timestamp - get messages after this time |

<CodeGroup>
```javascript Example: Fetch older messages
// Get most recent 50 messages
const recent = await fetch(
  `${BASE_URL}/v1/numbers/${numberId}/messages?limit=50`,
  { headers: { 'Authorization': `Bearer ${API_KEY}` } }
);

const messages = await recent.json();
const oldestMessage = messages.data[messages.data.length - 1];

// Get next 50 messages (older than the oldest we have)
if (messages.hasMore) {
  const older = await fetch(
    `${BASE_URL}/v1/numbers/${numberId}/messages?limit=50&before=${oldestMessage.receivedAt}`,
    { headers: { 'Authorization': `Bearer ${API_KEY}` } }
  );
}
```
</CodeGroup>

<Tip>
  **Best Practice:** Always check `hasMore` before fetching the next page. Use cursor-based pagination (before/after) for time-ordered data as it's more efficient than offset-based pagination.
</Tip>
